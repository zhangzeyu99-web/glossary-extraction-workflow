# Source-Only Glossary Delivery Retrospective

## 适用场景

当项目目录里只有中文源文语言表，暂时没有英文译文列时，使用本流程先完成术语提取、预翻译、分类整理和标准命名交付。

典型输入：

- 主语言表：包含 `ID / 内容 / 中文，用于导出 / 是否翻译`
- UI 语言表：包含 `ID / 内容 / 中文，用于导出 / 是否翻译`
- 可能存在辅助 sheet，例如错误码表、系统提示表

核心原则：

- 不把客户原始语言表提交到仓库
- 输出写回用户指定目录
- 源文-only 阶段保留 `ID / CN / EN / EN2` 结构
- 分类整理阶段将 `分类` 放在术语表最后一列
- 同类术语放在一起，便于翻译、LQA 和术语管理员筛选

## 交付阶段

### 1. 已提取

目标是从源文中提取术语候选。

输出字段：

```text
ID / CN / EN / EN2
```

处理口径：

- `CN` 从源文列提取
- `EN / EN2` 在没有英文译文时允许为空
- 如果 `curated_terms.json` 中已有人工规则，可以自动补入 `EN / EN2`
- 低风险的一次性整句不进入核心术语表

推荐命名：

```text
项目名-已提取-YYYYMMDD.xlsx
```

### 2. 预翻译

目标是给已提取术语补一版可审校英文。

输出字段：

```text
ID / CN / EN / EN2
```

处理口径：

- `EN` 填标准译法
- `EN2` 只填常见备选、按钮适配、属性缩写或游戏内另一套稳定用法
- 人名、系统名、资源名优先保持全局一致
- `##1 / ##2` 等占位符必须保留，且顺序不能错

推荐命名：

```text
项目名-预翻译-YYYYMMDD.xlsx
```

### 3. 已分类

目标是按游戏本地化术语库规格整理术语，便于后续审校和维护。

输出字段：

```text
ID / CN / EN / EN2 / 分类
```

处理口径：

- `分类` 放在最后一列
- 主表按分类排序，同类术语连续排列
- 统计 sheet 单独列出分类数量
- 说明 sheet 单独写明命名规则、状态、日期和审校风险

推荐命名：

```text
项目名-已分类-YYYYMMDD.xlsx
```

如果分类版同时已经包含预翻译，可以使用：

```text
项目名-预翻译-YYYYMMDD.xlsx
```

并在说明 sheet 标注“已按分类排序”。

### 4. 已审校

目标是人工确认术语译法和分类。

推荐命名：

```text
项目名-已审校-YYYYMMDD.xlsx
```

审校重点：

- `EN2` 是否真的需要保留
- `Legion / Alliance` 这类系统口径是否已经拍板
- `ATK / Attack`、`DEF / Defense`、`HP / Health` 等缩写是否符合游戏 UI
- 人名是否和角色设定一致
- 商品名是否适合商城展示长度

### 5. 已回灌

目标是把人工确认结果回写到规则层。

命令：

```bash
python scripts/import_curated_glossary.py /path/to/项目名-已审校-YYYYMMDD.xlsx \
  --curated-rules data/experience/curated_terms.json
```

推荐命名：

```text
项目名-已回灌-YYYYMMDD.xlsx
```

## 分类体系

默认大类：

- `核心系统/功能名`
- `UI操作动词`
- `战斗属性/数值`
- `资源/货币/材料`
- `道具/装备/礼包`
- `角色/人名`
- `社交/军团`
- `任务/活动/玩法`
- `状态/时间/进度`
- `建筑/地点`
- `聊天/账号/设置`
- `格式变量/数值表达`
- `通用名词/形容词`

分类原则：

- 系统入口、玩法入口、一级功能优先归入 `核心系统/功能名`
- 按钮、弹窗、菜单操作归入 `UI操作动词`
- 属性、伤害、战力、战斗机制归入 `战斗属性/数值`
- 货币、材料、积分、资源归入 `资源/货币/材料`
- 商城商品、礼包、装备、宝箱归入 `道具/装备/礼包`
- 军团、好友、聊天、成员体系按功能分别归入社交或聊天类
- 含 `##1 / ##2` 的词条优先归入 `格式变量/数值表达`

## QA 检查

交付前至少检查：

- workbook 能用 `openpyxl.load_workbook(..., read_only=True, data_only=True)` 复开
- 主表列顺序符合目标状态
- 主表行数和来源术语数一致
- `EN` 无空值，除非状态是 `已提取`
- `分类` 无空值，除非状态是 `已提取` 或 `预翻译`
- `##1 / ##2` 等占位符在 `CN / EN / EN2` 中保持一致
- 分类没有重复分段，同类术语应连续排列
- 输出文件在用户指定目录，不进入仓库

## Windows 路径注意事项

中文路径和中文文件名在部分 PowerShell 到 Python 的管道场景中可能被转成 `????`。

稳定做法：

- 用 PowerShell `Resolve-Path -LiteralPath` 解析目录
- 通过环境变量把目录传给 Python
- 一次性脚本保存为 UTF-8 文件后执行，避免长中文脚本通过管道传输
- 如写入失败，先使用 ASCII 文件名输出，再按最终命名规则生成交付文件

## 仓库维护边界

可以提交到仓库：

- 流程文档
- 分类体系
- 命名规则
- 脱敏 fixture
- 通用脚本能力

不提交到仓库：

- 客户原始语言表
- 真实项目交付表
- 含未上线内容的术语清单
- 一次性临时脚本
